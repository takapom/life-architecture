# ファクトチェックプレイブックの登録

## 目的
DAG エクスポート前に、各ソースアイテムを分類し、トレーサビリティのためにエビデンスを保持する。

## 判定
- valid: アクション可能。1つのノード `covers` にマッピングする必要がある
- already-fixed: `source_items` に保持するが、実装ノードからは除外する
- invalid: 監査のために `source_items` に保持する
- pending: 最終判断まで保留する

## エビデンス基準
- 常に `path:line` を引用する。
- ランタイム動作の場合、実行エントリパスを含める。
- 設定駆動の動作の場合、設定ファイルパスを含める。

## 登録ルール
1. `valid` アイテムは正確に1つの `covers` オーナーにマッピングする必要がある。
2. `pending` アイテムは `deferred_items` に存在する必要がある。
3. non-valid アイテムを `source_items` から削除しない。
4. DAG 分割を通じてソースアイテム ID を安定させる。
