# OMTA デザインシステム・インタラクションアダプター

メソドロジーファイルが正しい UX の方向性を定義した後に適用する。このファイルは OMTA がその方向性をどう表現すべきかを制約する。

## デザインシステムの SoT

- `packages/ui/src/styles/tokens.css` を共有トークンの SoT として扱う。
- `apps/app/src/app/styles/tokens.css` は共有トークンのオーナーシップを再定義せず、アプリローカルトークンを拡張する。
- テーマ初期化とテーマプロバイダーは `@omta/ui` に保持する。
- 2モード規約を維持する: `light` と `dark` のみ。
- シェルの差異はレイアウト、密度、装飾で表現し、プライベートなカラートークンフォークでは表現しない。

## ビジュアル言語

- 落ち着いたサーフェス、ワークスペースクロームトークン、控えめなシャドウ、トークン化されたアクセント装飾を優先する。
- パレットと強調はセマンティックトークン用語（surface、border、primary、success、warning、workspace chrome など）で記述し、場当たり的なカラーピックでは記述しない。
- 場当たり的なフォントスタックを導入する代わりに、既存のフォントとフォントトークンを使用する。
- グラデーションは、アクセントバーやヒーローオーバーレイなど、既にトークン化されていて控えめな場合のみ許可する。
- 汎用的な AI スタイルのヒーローダッシュボード、グロー過多のダーク UI、共有トークン規約をバイパスする一回限りのグラデーションを避ける。
- タスクが真に OMTA のビジュアルディレクションを変更する場合は、単一サーフェスにカスタムスタイリングをパッチする代わりに、関連するトークンまたは UX トーンの SoT を更新する。

## プリミティブの再利用

新しいマークアップを発明する前に、既存の共有コンポーネントを優先する:

- `PageHeader`
- `EmptyState`
- `AccentedCard`
- `Panel`、`PanelLoading`、`PanelError`
- `Modal`、`Sheet`、`ConfirmDialog`
- `DataToolbar`
- `Skeleton`
- `SectionLabel`、`Kicker`、`Badge`、`Stepper`

マーケティングサーフェスはワークスペースサーフェスよりも表現豊かにできるが、同じ共有テーマとトークン規約を消費する。マーケティングの差別化は主にコンテンツ構造、イラスト、構成、承認されたトークン使用を通じて行い、プライベートなトークンフォークでは行わない。

## OMTA 表現ルール

- ナビゲーションにはリンク、アクションにはボタンを使用する。
- ボタン形状のナビゲーションには `<Button asChild><Link /></Button>` を使用する。
- すべてのアイコンのみのボタンに `aria-label` を付与する。
- 破壊的アクションは確認またはアンドゥパスの背後に置く。
- 空状態はメインコンテンツエリアに表示する。空白のシェルを残さない。
- 長いテキストは意図的にトランケートまたはラップし、縮小が必要な flex 子要素には `min-w-0` を追加する。
- フィルター、タブ、ページネーションを URL 状態に反映する。
- バリデーションエラーはインラインで表示し、送信時に最初の無効なフィールドにフォーカスする。
- `prefers-reduced-motion` を尊重する。
- 明示的なプロパティのみをアニメーションする。`transition: all` を使用しない。
- モバイルレイアウトを壊すビューポート高さのショートカットの代わりに `h-dvh` または `min-h-dvh` を使用する。
- シート、トースト、スティッキーバー、固定モバイルアフォーダンスではセーフエリアインセットを尊重する。

## コピールール

- ユーザー向けテキストは i18n メッセージカタログに保持し、インライン TSX 文字列にしない。
- 日本語と英語のキーパリティを維持する。
- リポジトリローカルの用語とローカライズされたラベルを優先し、散在する英語 UI テキストを避ける。
- コピーは装飾的ではなく、運用的、直接的、コンテキスト対応にする。

## ローディングとルート構成

- 汎用的なローディングファクトリーではなく、ページ固有の `loading.tsx` スケルトンを実装する。
- 子が実際にサスペンドする場合にのみ `Suspense` を追加する。
- App Router ファイルはビジネスロジックではなく構成に集中させる。

## アダプターレビュー質問

- これは現在のオーナーサーフェスに属するか？
- 新しいものを作る前に既存のプリミティブまたはトークンを再利用しているか？
- OMTA のコピー、アクセシビリティ、モーションルールを保持しているか？
- 明示的な承認や対応する SoT 更新なしに、OMTA のトークン/テーマ/UX 規約からドリフトしていないか？
