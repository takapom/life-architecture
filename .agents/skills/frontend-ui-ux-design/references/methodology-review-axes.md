# インタラクション・情報デザイン レビュー軸

リポジトリ固有の実装ルールよりも先に、これらの軸を適用する。これがスキルの再利用可能なコア。

## ノーススターテスト

- `理解性`: ユーザーは自分がどこにいるか、今何が重要か、システムがどの状態にあるかを判別できるか？
- `意思決定`: ユーザーはすべてを読まなくても最善の次のアクションを特定できるか？
- `実行`: ユーザーは低摩擦、低曖昧性、低リスクで行動できるか？
- `リカバリ`: ユーザーはミスを検知し、結果を理解し、安全に回復できるか？
- `発展性`: このパターンは、より多くの状態、データ、権限、長いコンテンツでも機能し続けるか？

## レビュー軸

### 目的の明確性

- サーフェスには明確な役割と明確な成功条件があるべき。
- 失敗シグナル: 混在した目的、曖昧な見出し、主な役割にマッピングされないアクション、目的を不明瞭にする装飾的コンテンツ。

### アクション階層

- プライマリ、セカンダリ、ターシャリ、破壊的アクションが視覚的・構造的に競合してはならない。
- 失敗シグナル: 競合するプライマリ CTA、重複した送信アクション、安全なアクションとリスキーなアクションの均等な強調、必要になる前に表示されるコントロール。

### 情報階層

- ユーザーは意思決定を支援する順序でサーフェスをスキャンできるべき。
- 失敗シグナル: テキストの壁、繰り返されるヘルパーコピー、弱いセクションラベル、隠されたキーデータ、逐次読解を強いるレイアウト。

### 認知負荷管理

- ユーザーが一度に読む、覚える、比較する、推論する量を減らす。
- 失敗シグナル: 同時選択肢が多すぎる、説明コピーが多すぎる、過負荷なフォーム、認識ではなく記憶を要求する画面。

### 段階的開示

- 次の正しいアクションに必要な最小限を表示し、詳細はオンデマンドで公開する。
- 失敗シグナル: デフォルトで表示される詳細設定、ハッピーパスを支配する例外処理、基本的な意図が確立される前の長い説明。

### 状態の可読性

- 現在の状態、保留状態、成功、エラー、空状態、ロック状態が可視的かつ区別可能であるべき。
- 失敗シグナル: 空白のシェル、曖昧なローダー、隠された副作用、不明確な選択状態、確認後に何が変わるか説明しないアクション。

### エラー防止とリカバリ

- まず高コストなミスを防止し、次にリカバリを明示的にする。
- 失敗シグナル: 確認やアンドゥなしの破壊的アクション、遅すぎるバリデーション、次のステップのないエラー、隠された結果を伴う不可逆アクション。

### 一貫性とシステム活用

- 実証済みのパターン、コンポーネント、命名、スペーシングを再利用し、ユーザーが既存知識を転用できるようにする。
- 失敗シグナル: 一般的なアクション向けの独自コントロール、一貫性のない CTA 配置、プライベートなビジュアル言語、類似サーフェス間のパターンドリフト。

### アクセシビリティと耐障害性

- サーフェスはキーボード操作、スクリーンリーダー、長い翻訳、狭いビューポート、モーション縮小、非典型的なデータでも機能すべき。
- 失敗シグナル: ラベルなしのアイコンのみのコントロール、テキストのクリッピング、モーションのみのフィードバック、不適切なフォーカス動作、長いコンテンツでのレイアウト崩れ。

### 拡張性

- 選択した構造は、将来の状態や新しい要件を書き直しなしで受け入れられるべき。
- 失敗シグナル: ハードコードされたハッピーパス、空/エラー/権限状態の余地がないレイアウト、今日のデータ形状でしか機能しないパターン。
