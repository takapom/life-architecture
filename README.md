# 人生アーキテクチャ診断 Webアプリ：要件定義書

## 1. プロダクト概要

- **コンセプト**: ユーザーの人生を「ソフトウェアアーキテクチャ」に見立てて診断・可視化するエンタメ型自己分析ツール
- **ビジョン**: 「自分の過去と現在をメタ認知する楽しさ」と「アーキテクチャ設計の基礎学習」を同時に提供する
- **ターゲット**: エンジニア（初学者〜シニア）、およびシステム設計の概念に興味がある人

---

## 2. ユーザー体験 (UX) フロー

1. **ランディング**: ターミナル風、またはクリーンなエディタ風UIでユーザーを迎え入れる
2. **現在診断 (Current Status)**: まずは「今の自分」を診断し、即座に価値を提供する
3. **歴史探索 (Migration History)**: 「なぜ今の構成になったのか？」を解明するため、過去（高校・大学・新卒）のデータを入力
4. **最終レポート**: 現在と過去を比較した「システム構成遷移図」と「リファクタリング計画」を提示
5. **シェア**: SNS（X/Zenn/Qiita）への放流（README.md形式や画像出力）

---

## 3. 主要機能 (Functional Requirements)

### ① AI診断エンジン（Gemini 1.5 Flash 活用）

- **自然言語処理**: ユーザーのラフなエピソードを技術用語にマッピングする
- **メタファー変換**: 「失恋＝セッション切れ」のような独自の人生・技術変換辞書に基づいた解説生成

### ② アーキテクチャ可視化

- **Mermaid.js**: 自動生成されたコードにより、人生の構成図をグラフ表示
- **特性チャート**: レーダーチャートによる「俊敏性」「可用性」「信頼性」等のスコアリング

### ③ 診断フェーズ管理

| フェーズ | アーキテクチャ傾向 |
|----------|-------------------|
| 高校時代 | モノリス、中央集権、密結合などのレガシー構成 |
| 大学時代 | マイクロサービス、陽キャAPI、分散処理、多言語（キャラ）対応 |
| 現在     | クラウドネイティブ、サーバーレス、UI負債、バックエンド最適化 |

---

## 4. 出力サマリーの標準構成

各フェーズの診断結果は、以下の4項目を必ず含む。

| 項目 | 内容 |
|------|------|
| アーキテクチャ名 | 技術用語を冠したキャッチーな名称（例：`Tennis-Centric Monolith`） |
| 構成図 (Diagram) | Mermaid.js等によるコンポーネント図 |
| 特性スコア | レーダーチャートによる数値評価 |
| 解説コメント | AIによるメタファーを交えた人生解説 |
