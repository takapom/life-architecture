# 要件定義

## UXフロー

```
ランディング
  ↓
現在の診断（8問）
  ↓
診断中（loading）
  ↓
診断結果（現在）
  ↓
「過去も診断する」ボタン ← 任意・1回のみ
  ↓
過去フェーズ作成
  プリセット：[ 学生時代 ] [ 新卒 ]
  カスタム：自由入力（例：20代前半、転職前 など）
  ↓
過去の診断（8問）
  ↓
診断中（loading）
  ↓
変遷図（現在 + 過去の比較）
  ↓
シェア（モックボタンのみ設置・仕様は実装後に決定）
```

### 補足

- 現在の診断と過去の診断は独立したフロー
- 過去フェーズの追加は1つのみ（複数追加不可）
- 「過去も診断する」はスキップ可能（現在の診断結果のみで完結できる）

## ヒアリング設計

### 基本方針

- 各フェーズ（現在・過去）につき8問の質問に答えてもらう
- 回答形式は**自由記述**（「わからない」も有効な回答として許容する）
- トーンは**カジュアル**
- 全問**必須回答**
- **1問ずつ表示**（回答して次の質問へ進む形式）

### 質問一覧

| # | 質問 | 対応するアーキテクチャ概念 |
|---|------|--------------------------|
| 1 | 最近、一番時間とエネルギーを注いでることって何？ | コアシステム / メインワークロード |
| 2 | 人との関わり方を教えて。よく頼る人や頼られる場面ってある？ | 結合度 / 依存関係 |
| 3 | 予想外のことが起きたとき、どう動く？エピソードあれば。 | 耐障害性 / レジリエンス |
| 4 | 「なんか変えたいな」「直さないとな」って思ってることある？ | 技術的負債 |
| 5 | 新しいことへの挑戦、どのくらいのペースでやってる？ | デプロイ頻度 / 変化への適応 |
| 6 | 忙しさのピークが来たとき、どうやって乗り越えてる？ | スケーラビリティ / 負荷耐性 |
| 7 | 自分の調子がいい・悪いって、どうやって気づく？ | モニタリング / 自己観測 |
| 8 | 今の自分を直感で一言表すと？ | セルフブランディング / ラベリング |

> Q8 は「ここまで振り返った上での一言」として最後に配置。AIが名前を生成する際の重要なヒントとなる。

## 診断結果の出力

### アーキテクチャ名の命名規則

**メジャーなアーキテクチャ用語 + エピソードから抽出したキーワード**の組み合わせ。響きとキャッチーさを重視する。

```
例：
  Tennis-Centric Monolith
  Captain-Driven Monolith
  Solo-Optimized Batch Processor
  High-Throughput Social Mesh
  Hobby-Scattered Microservices
```

### 出力サマリーの構成

各フェーズの診断結果は以下の4項目を含む。

| 項目 | 内容 |
|------|------|
| アーキテクチャ名 | メジャー用語 + 個人キーワードの組み合わせ名称 |
| 構成図 | コンポーネント図（React Flowで可視化） |
| 特性スコア | レーダーチャートによる数値評価（俊敏性・可用性・信頼性 等） |
| 解説コメント | AIによるメタファーを交えた人生解説 |
